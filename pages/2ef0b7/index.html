吾不该止步于此！面试记录原创

# 吾不该止步于此！面试经典话术

# 自我介绍

自己的专业是软件工程，有过一段实习经历，在工作时，能够独立完成需求，通过这次工作也让自己能够从0到1搭建一个vue3+ts的项目，自己学习前端的过程，刚开始主要通过观看视频，为了加深理解，会阅读官方文档以及文档中的举例，后面会学习一些github上的优秀项目，比如element-ui学习如何封装组件，ruoyi如何实现前后端联调以及权限控制，antd-design-vue如何实现项目国际化和主题切换

对自己的期望是能够运用自己的专业知识，为项目团队创造自己的价值，并且这个过程中能够提升自己的技术

# 职业规划

# 为什么选择前端

大学期间主要开设后端课程，之所以选择前端，是因为大学期间开设过一门网页设计的课程，利用html+css写了一些页面，自己觉得比较神奇，也比较喜欢编写前端代码渲染成页面这种直观视觉感受，感觉很有成就感，后面兴趣也越来越浓，就选择了前端，开始了学习前端的道路

# 是否有前端引路人

# 如何学习，构建自己的前端体系

  • 前期的话是根据视频学习，慢慢地了解到知识的体系
  • 中间为了理解更加深刻会看一下文档以及文档中的例子
  • 后期主要在github上学习一些优秀项目
    • 比如element源码理解组件封装思想、ruoyi项目理解前后端如何联调和权限控制、antd-design-vue理解前端如何做国际化和主题切换

# 遇到问题会如何解决

  • 遇到问题，一般会先锁定出现问题的前提条件，再根据这个前提条件确定问题关键词，再利用关键词了解网上的一些知识，比如在stackoverflow等网站，尝试试错解决方案，如果有空余时间，再确认哪种方案比较好
  • 在一体化项目的时候，遇到一个问题，在工程列表中有一个工程设置按钮，点击工程设置会显示工程详情，如果用户没有修改详情的话，点击返回直接跳到列表，如果修改了，需要有一个保存提醒
  • 这里需要判断刚开始进入的对象和点击按钮的那个对象是否相等,刚开始只想到了需要利用本地存储先保存，点击之后在取出来与当前对象做比较，之后在对象比较这个卡住了，想过自己封装个函数，但是想到对象属性数据可能会有多种，封装起来会比较复杂，之后了解到JSON.stringify这个方法，可以将对象转成字符串再比较，之后这个问题得到完美解决

# 对第一份工作的期待

  • 能够运用我自己的专业知识和技能，创造自己的价值，因为这样能够满足我的成就感
  • 在满足领导的要求的同时，还能够有技术水平上的提升，解决更多的问题

# 对找工作考虑的因素有哪些

  • 一个是团队氛围，这样能够激发我的主观能动性
  • 一个是领导能够严格要求团队人员，这样才能让团队和个人双向进步

# 工作的心得

前端的工作一般是从后端拿数据渲染到页面上，直接面向用户，无论从页面效果和交互性来说，如何做到一个功能能让用户使用起来非常顺心，拥有更好的体验，是我们需要考虑的

# 对自己的缺点认识

自我感觉，我的执行力不错，在我实习的时候，能够在规定时间内完成领导的安排，也正因为这样，自己更多的是在行动上的表现,自己深度思考就相对少一些，所以这个是我需要加强的

# 为什么选择我们公司

因为我了解到公司是做点电商、物联网等技术的，无论是在运营团队还是在技术团队都有一定的实力，本身自己也是倾向在互联网行业发展的，希望能够在公司的带领下，完成自己的价值

# 对加班的看法

加班对于互联网行业来说本来就是常态，加上自己没有负担，如果需要加班，我会做到义不容辞，同时自己也尽力提高自己的工作效率

# 遇到难以相处的同事

自己会以大局为重，即便在一段时间内，周围同事对自己不理解，我也会先做好本职工作，虚心向他们学习，相信他们会看到自己的努力的

# 如何对待别人的批评

沉默是金，在大家情绪化的情况下，不必说什么，否则会让情况更糟，但是我会先接受别人建设性的批评，最后我会等大家冷静下来之后再讨论，是人的问题还是流程的问题，最后加以改进和优化

# 如何看待自己的失败

我相信人不是十全十美，但是我相信我能够改正我的错误

# 你觉得这个工作有什么困难

工作中的困难在所难免，但是我相信只要有专业的团队、良好的合作精神，这些都是可以克服的

# 你还有什么想问的

  • 负责的项目，技术栈
  • 提高转正率需要达成什么指标
