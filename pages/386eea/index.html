吾不该止步于此！大学本科答辩原创

# 吾不该止步于此！大学本科答辩

# 答辩注意

  • 讲解时间为10分钟，答辩教师提问时间5分钟
  • 答辩过程中，答辩秘书需要将首页截屏保存，请大家演示PPT的过程中将首页停留30秒再进行讲解
  • 答辩过程中需要全程开启摄像头，保证摄像头和PPT同时呈现
  • 答辩记录表：由答辩秘书帮同学们记录老师提问的问题，在你们自己答辩结束后，答辩秘书将问题发到你们企业微信里，自己将答案和问题填写到记录表中，之后立即发回给答辩秘书统一收集
  • 答辩记录表由答辩组收集以后统一管理，你们不用打印了，但是电子版表格必须全部返回给答辩秘书！！切记
  • 展示论文1分钟，讲ppt花2分钟，演示系统5分钟，这就8分钟了，可以停了

# 介绍

  • 各位老师们，上午好，我是软工2班的曾慧清，答辩老师是郭赛克老师，论文答辩的题目是基于SpringBoot的软考在线培训系统的设计与实现。系统是基于Spring Boot和Vue.js的前后端分离系统，系统分为学员端系统和管理端系统
  • 首先学员端系统，登录进来，学员的培训过程有理论知识学习-做题实践-模拟考试进行查漏补缺
  • 首先，理论知识学习，通过选择不同类型的课程进行学习，这里点击课程章节可以进行观看视频
  • 做题实践,可以通过题目训练进行，根据不同科目、年份、年度的条件进行选择试题，在训练题目这里有个分页功能，以及在答题时会有一个实时提示
  • 模拟考试，就是学员根据条件进行选择，在查看详情时需要填写个人信息才能进行模拟考试
  • 除了以上功能还有社区讨论和咨询留言，社区讨论目的是为了让学员可以针对某一问题进行讨论，发散思维，提高自己的只是维度，间接提高培训的效率
  • 咨询留言的话，通过发送邮箱的方式添加咨询人，点击资讯列表进行咨询问题
  • 管理端系统的话，这里演示讲师的课程管理和试卷管理
  • 课程管理，讲师新增课程，填写信息，进行提交，经过管理员审核之后，讲师可以选择上架
  • 设置试卷可以通过手动添加，和文件导入，这里演示文件导入，需要下载模板，之后填充内容，在选择文件导入试卷，导入后可以查看

# 结尾

  • 最后，感谢各位老师们balabala

# 准备材料

  • 线上答辩的同学，资料寄给我，或者找到学校答辩的同学帮忙归档后，交给我
  • 【纸质版归档材料】在系统提交个人签名，下载带电子签名的各项文档：
    • 任务书（系统导出，调整格式后双面打印）
    • 开题报告（系统导出，调整格式后双面打印）
    • 装订成册的毕业论文终稿（封面到目录单面打印，正文第一章开始双面打印），用答辩后，完全没问题的版本！！
    • 毕业论文（设计）评分手册封面扉页（指导教师补充其余部分）
    • 毕业论文（设计）检测报告（原文对照报告pdf版）
  • 【电子版归档材料（光盘刻录）】
    • 任务书
    • 开题报告
    • 论文日志（3篇以上）
    • 电子版论文初稿（含有批改记录的两版）、终稿（答辩后按要求修订版）
    • 毕业论文（设计）检测报告（原文对照报告pdf版或全部版本）
    • 答辩PPT：项目支撑材料（源码、系统演示视频、设计图纸、数据表等）
