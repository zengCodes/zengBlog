# 吾不该止步于此！追逐新鲜感的道路越走越远系列之Vue3

# vite区别于webpack

  • webpack在开发环境和生产环境中都需要进行打包，如果项目模块依赖特别多，打包起来会特别慢，所以运行起来也会特别慢，影响开发效率
  • vite解决开发环境下运行慢的问题，具有以下几个特点：
    • 运行时无需编译，检查到对应的页面会加载编译对应的代码
    • 基于EsBuild预构建依赖
    • 支持热替换
    • 生产环境需要进行打包，通过rollup打包编译成es module
    • 由于是新技术，可能生态没有webpack好，关于文件处理和压缩代码等功能支持没有那么好

# vue3区别于vue2（为什么选择vue3）

  • 更小
    • 引入了tree-shaking，实现真正的按需加载
  • 更快
    • 在响应式上从Object.defineProperties改成Proxy+Reflect
    • 在diff算法做了优化，将将要改变的元素进行标记，在diff时只需要找这些标记就行，提高diff效率
    • 静态提升，将静态节点进行创建后提升到全局作用域下，后续使用时直接复用
    • 事件监听缓存，对事件绑定的函数缓存起来，后续使用直接从缓存中拿，不用再次创建
  • 更友好
    • 在兼容原本option Api的同时还推出了Composition Api
    • 对于Composition Api，在逻辑组织和复用上有很大优势，也利于tree-shaking
    • 由于Composition Api是函数式编程，所以对ts支持友好，有利于类型的支持
    • 去除vue2中this，避免this指向不明确的问题
  • 利于源码维护
    • 在源码中，采用Monorepo模式进行管理，实现在同一个package下管理多个模块的代码，模块依赖性更强，也利于模块代码的复用，减少代码冗余

# Pinia区别于Vuex

  • 友好的支持ts
  • 保留原有的state、getter、actions,去除了mutations
  • 更改状态一般可以通过state.属性来修改或者通过actions修改，官方推荐在actions中修改状态值
  • 避免了嵌套模块，使用单文件store来定义模块的状态管理，利于代码拆分和开发效率
