# 吾不该止步于此！刷够多少题才够系列之JavaScript

# ES

# 基本数据类型

  • Number
  • String
  • null
  • undefined
  • boolean
  • Symbol
    • symbol类型不能使用new关键字
    • 一般用于定义对象属性唯一性，私有属性
    • 必须使用特定方法才能识别，Object.getOwnPropertySymbols(obj)返回一个数组
    • Symbol()==Symbol()返回false

# let、const、var

  • let声明变量、const声明的是常量，并且使用前必须初始化，不能修改
  • let、const存在暂存性死区，不会出现变量提升的情况
  • var不存在暂存性死区，会出现变量提升的情况

# this指向

  • this一般指向window全局对象
  • 其他特殊情况指向调用者对象，可以利用apply、call、bind改变this指向
    • apply接收参数数组
    • call、bind接收参数列表
    • call、apply会立即执行，bind改变this指向之后返回新函数，不会立即执行

# 数组方法

# 创建数组

  • arr = new Array()和 arr= []
  • 扩展运算符：[...arr]
  • Array.of：不支持类数组
  • Array.form：支持类数组
  • Array.prototype.fill：参数构成有填充内容，起始位置，结束位置，改变原数组

# 堆栈方法

  • arr.pop()：从尾部删除-----返回删除的项目
  • arr.push()：从尾部增加-----返回增加后的数组长度
  • arr.shift()：从头部删除----返回删除的项目
  • arr.unshift()：从头部增加----返回增加后的数组长度

# 字符串方法

  • arr.toString()：返回逗号隔开的字符串
  • arr.join()：连接符连接元素，数组元素合成字符串，参数为连接符

# 删除元素

  • arr.splice()：参数构成：起始位置，删除元素个数，增加元素，返回一个被删除的数组项目，会改变原数组
  • arr.slice()：截取元素 左闭右开，返回新数组，不会改变原始数组

# 排序方法

  • arr.sort()：数组排序，返回原数组，会改变原数组，排序原理是根据元素UniCode编码排序，底层用的是插入排序和冒泡排序
  • arr.reverse()：数组反转，返回原数组，会改变原数组

# 寻找下标方法/含有某个元素

  • arr.indexOf：返回元素首次出现下标，如果没有返回-1，不能够判断NaN
  • arr.lastIndexOf：返回元素最后一次出现下标，如果没有返回-1，不能够判断NaN
  • arr.includes()：返回布尔值，能够判断NaN
  • Array.prototype.at:如果下标小于0,可以从末尾开始找

# 数组连接

  • arr.concat()：不能处理多维数组

# 数组遍历

  • forEach
  • map
  • filter
  • some
  • every
  • reduce
  • arr.keys
  • arr.values：返回可迭代数组
  • arr.entries：返回可迭代键值数组
  • for in
    • 如果是数组，会遍历数组下标，但是下标属于字符串
    • 会遍历原型链上的可枚举属性
    • 更适合遍历对象
  • for of
    • 属于Es6方法
    • 遍历数组元素值
    • 更适合遍历数组
    • 相比forEach能够正确响应break、continue语句
    • 不能遍历对象

# 数组扁平化

  • flat: 接收一个参数，表示扁平化层数，默认为1
  • flatMap: 将数组经过处理后展平

# 对象方法

  • Object.is
  • Object.assign：拷贝源对象的可枚举属性
  • Object.create：创建新对象
  • Object.freeze: 冻结一个对象，不允许修改

# Object.is区别普通==

  • ==会有隐式类型转换，而Object.is不会
console.log(-0 == +0); // true
console.log(Object.is(+0, -0)); // false
console.log(Object.is(NaN, NaN)); // true

# ES6+

# 异步请求发展

# ajax

  • 使用XMLHttpRequest对象进行请求
  • 容易产生回调地狱的问题

# promise

  • 一个promise拥有三种状态，pending、fulfilled、rejected
  • 一个promise一次只能修改一次状态
  • 支持链式调用
  • all、any、allSettled、race、resolve、reject
    • all如果失败就返回
    • any返回第一个成功的promise
    • 返回所有的promise
    • race返回第一个promise

# Generator

  • 函数名旁边有*
  • 函数内部用yield定义不同的状态
  • 返回一个可迭代对象

# async

  • 返回一个Promise对象
  • 返回值是函数返回值或者抛出异常的信息

# await

  • await后面接的是一个表达式
  • 返回值是一个promise等待的结果或者是表达式返回值

# 为什么使用async/await

  • 目的是为了简化代码，能更简洁的使用基于Promise的异步行为，而不用刻意的使用Promise的链式调用

# 数组方法

console.log(Array.isArray([])); // true
console.log(new Array(5).fill(1)); // [1, 1, 1, 1, 1]
console.log(new Array(2)); // [ 空属性*2 ] 空属性不等于undefined
console.log(Array.of(2)); // [2]
let str = "foo"
console.log(Array.of(str)); // ['foo']
console.log(Array.from(str)); // ['f', 'o', 'o']
let array = [1, 2, [3, 4, [5, 6]]]
console.log(array.flat(2)); // [1, 2, 3, 4, 5, 6] 将嵌套的数组展开，也就是实现数组扁平化，参数默认为1
console.log([1, 2, 3, {},
  [], NaN
].includes(NaN)); // true

# new Array()、Array.of、Array.from区别

  • 当需要传一个值到数组时，即new Array(2)，创建的是一个拥有两个empty（空属性）的数组，解决这个问题就出现了Array.of
  • Array.from能够把类数组转为数组

# includes和indexOf区别

  • includes返回布尔值，indexOf返回元素下标
  • includes能够判断NaN,而indexOf不能

# 数组遍历方法区别

  • map是数组的映射，不会改变原来数组，返回一个新数组
  • filter是过滤数组，不会更改原来数组，返回判断条件为true的每一个item组成的新数组
  • forEach是遍历数组，没有返回值
  • some返回布尔值，至少一个元素满足条件才返回true
  • every返回布尔值，全部满足条件才返回true
  • reduce返回遍历完后的数组，接收四个参数，包含上一次循环返回值和当前遍历元素，数组下标，当前遍历数组

# forEach区别普通for循环

  • forEach不能响应return/break/continue,for可以
  • forEach不能改变原数组，for可以
  • forEach对于数组元素为空的话（稀疏数组）直接跳到下一次数组元素不为空的循环
  • forEach跳出循环的话，需要抛出异常throw error
let array = [1, 2, 3, 4]
  array.forEach(item => {
     if (item % 2 == 0) {
      delete item
  }
})
console.log(array); // [1, 2, 3, 4]
for (let i = 0; i < array.length; i++) {
   if (array[i] % 2 == 0) {
      delete array[i]
   }
}
console.log(array); // [1, 空白, 3, 空白]
let index1 = 0
for (let i = 0; i < array.length; i++) {
   console.log(array[i]); // 1 undefined 3 undefined
   index1++
}
console.log("for循环后的index：" + index1);
let index2 = 0
array.forEach(item => {
  console.log(item); // 1 3
  index2++
})
console.log("forEach循环的index:" + index2); // 2

# 函数

# 函数声明和函数表达式

  • 函数声明享受函数提升
  • 函数表达式享受变量提升

# arguments对象

  • 函数传递参数的类数组对象
  • arguments.callee实现递归

# 剩余参数

# 函数的length

  • 返回函数具有默认值参数中前面的参数个数，默认为0

# arguments对象和剩余参数的区别

  • arguments不是数组，不能使用数组方法，而剩余参数是数组，可以使用数组方法
  • arguments包含函数所有的实参，而剩余参数只包含对应形参的实参

# 箭头函数

  • this指向定义它的那个对象
  • 没有构造函数，不能使用new关键字
  • 没有arguments对象

# set、map、weakSet、weakMap

  • set表示一个没有重复元素的集合
  • map表示一个具有键值对的有序集合
  • weakSet、weakMap都是弱引用，在没有引用的情况下会进行垃圾回收
  • weakMap只能存键为对象的值和weakSet是对象的集合

# 事件模型-DOM事件流的三个阶段

  • 事件模型包含冒泡、捕获、目标三个阶段
  • 冒泡是由下至上，捕获是由上至下，目标阶段是事件触发时侯
  • 事件委托
    • 如果一个父元素的全部子元素都需要绑定事件，可以把事件绑定到父元素
    • 利用的是事件冒泡的原理
    • target返回触发事件的元素，不一定是绑定事件的元素
    • currentTarget返回的是绑定事件的元素
//阻止浏览器的默认行为 
function stopDefault( e ) { 
   //阻止默认浏览器动作(W3C) 
  if ( e && e.preventDefault ) 
     e.preventDefault(); 
  //IE中阻止函数器默认动作的方式 
  else window.event.returnValue = false; 
  return false; 
}
// 阻止事件冒泡
function stopBubble(e) { 
    //如果提供了事件对象，则这是一个非IE浏览器 
    if ( e && e.stopPropagation ) 
    //因此它支持W3C的stopPropagation()方法 
        e.stopPropagation(); 
    else 
    //否则，我们需要使用IE的方式来取消事件冒泡 
        window.event.cancelBubble = true; 
}

# 作用域

  • 定义：声明变量或者方法发挥作用的区域
  • 作用域链
    • 函数执行过程中，会优先查找自己的内部变量
    • 否则，会依次在作用域下依次向上查找

# JavaScript延迟加载

  • async，表示js加载、执行过程于dom渲染过程异步进行，不过在js加载完毕后会立即执行，可能不会按照顺序执行
  • defer，表示js加载过程于dom渲染过程异步进行，不过js的执行过程需要等到dom元素渲染完成之后，且在domcontentload事件之前，按顺序执行
  • 动态创建script标签，在DOMContentLoaded事件触发时，添加到页面
  • setTimeOut延迟代码执行

  • 内核差异

    • Chrome webkit

# 前端模块化

  • 前端模块化是指把前端文件看成独立的模块，比如js、css文件，利于项目的维护，提高耦合度
  • 由于模块之间会相互依赖，这样就出现了ES6规范、AMD、CMD规范，以及文件编译和模块打包工具webpack、vite
  • 比较常用的是ES6规范，通过export、import导入、导出模块
  • export和export default区别
    • export导出多个对象，export default只能导出一个对象
