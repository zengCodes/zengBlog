# 吾不该止步于此！如何搭建一个 VUE3 项目? 谁看谁不迷糊

# 脚手架安装

npm init vite project_name -- --template vue
选择vue+ts项目
确认之后进入根目录，下载依赖
1
2
3

# 初始化

  • 安装vue-router

    • 在路由文件下，使用createRouter创建路由实例对象，使用history模式，createWebHashHistory
    • 导出路由对象
    • 在main.ts文件下导入，并且使用app.use(router)挂载到vue实例上

  • 安装状态管理vuex

    • 在状态管理文件下，使用createStore创建vuex实例对象
    • 导出状态管理对象
    • 在main.ts文件下导入，并且使用app.use(store)挂载到vue实例上

  • 配置路径别名

    • 在vite.config.js文件下，配置resolve.alias

  • 配置script中name属性

    • yarn add vite-plugin-vue-setup-extend -D

# 其他

  • 使用node内置模块
    • @types/node
  • svg
    • vite-plugin-svg-icons
  • 自动导入vue的api
    • unplugin-auto-import
  • 自动导入组件
    • unplugin-vue-components
  • yapi自动生成接口和响应
    • yapi-to-typescript

# question

  • vue3-setup中获取dom节点的ref

    • onMounted中可以获取dom节点
    • 获取dom节点并且修改style

  • 父组件获取子组件的属性或者方法

    • 子组件需要通过defineExpose把属性或者方法暴露出去

  • 声明数据时，对于ref数据需要变量.value访问数据，对于reactive数据不能重新赋值，否则会失去响应式

  • 移动端ui组件库的选择

    • vant支持vue3
    • 同时vant组件的占用大小相对较小

  • vant组件按需引入

    • 有些组件需要单独引入(Toast、Dialog)

  • 更新依赖

    • npm install -g npm-check-updates
    • ncu -u
    • npm i

  • typeerror: cannot read properties of null (reading 'contains')
